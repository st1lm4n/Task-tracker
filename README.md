# Task Tracker

Система для отслеживания задач сотрудников с возможностью назначения, контроля выполнения и анализа загруженности.

## 🚀 Функциональность

- **Управление пользователями**: CRUD операции для сотрудников с ролями (admin, manager, employee)

- **Управление задачами**: CRUD операции с поддержкой иерархии задач (родительские/дочерние)

 - **Специальные отчеты**:

   - 📊 Занятые сотрудники - список сотрудников, отсортированный по количеству активных задач

   - ⚠️ Важные задачи - задачи, от которых зависят другие, с рекомендациями по назначению

- **REST API**: Полноценное API с JWT аутентификацией

- **Веб-интерфейс**: Интуитивный интерфейс на Django Templates + Bootstrap

## 🛠 Технологии
- **Backend**: Python 3.11, Django 4.2.7, Django REST Framework

- **База данных**: PostgreSQL (production), SQLite (development)

- **Аутентификация**: JWT tokens

- **Контейнеризация**: Docker, Docker Compose

- **Документация API**: Swagger/OpenAPI через drf-spectacular

- **Frontend**: HTML5, Bootstrap 5, Django Templates

## 📦 Установка и запуск

### Вариант 1: Запуск через Docker (рекомендуется)

1. Клонируйте репозиторий:

```bash
git clone <repository-url>
cd task-tracker
```

2. Создайте файл .env на основе примера:

```bash
cp .env_example .env
```

3. Запустите приложение:

```bash
docker-compose up -d --build
```

4. Приложение будет доступно по адресу: http://localhost

### Вариант 2: Локальная разработка

1. Установите зависимости:

```bash
cd app
pip install -r requirements.txt
```

2. Настройте переменные окружения или используйте SQLite по умолчанию

3. Выполните миграции:

```bash
python manage.py migrate
```

4. Запустите сервер разработки:

```bash
python manage.py runserver
```

5. Приложение будет доступно по адресу: http://127.0.0.1:8000

## 🔧 Настройка окружения

Создайте файл `.env` в корне проекта:

```env
# Django
SECRET_KEY=your-secret-key-here-change-in-production
DEBUG=1

# Database (для Docker)
POSTGRES_DB=task_tracker_db
POSTGRES_USER=task_tracker_user
POSTGRES_PASSWORD=tasktrackerpassword

# Hosts
ALLOWED_HOSTS=127.0.0.1,localhost,0.0.0.0
ORIGINS=http://127.0.0.1:8000,http://localhost:8000,http://0.0.0.0:8000

# CORS
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000
```

## 📋 API Endpoints

### Аутентификация

- `POST /api/auth/token/` - Получение JWT токена

- `POST /api/auth/token/refresh/` - Обновление JWT токена

### Пользователи

- `GET /api/users/` - Список пользователей

- `POST /api/users/`- Создание пользователя

- `GET /api/users/{id}/` - Информация о пользователе

- `PUT /api/users/{id}/` - Обновление пользователя

- `DELETE /api/users/{id}/` - Удаление пользователя

- `GET /api/users/me/` - Информация о текущем пользователе

### Задачи

- `GET /api/tasks/` - Список задач

- `POST /api/tasks/`- Создание задачи

- `GET /api/tasks/{id}/` - Информация о задаче

- `PUT /api/tasks/{id}/` - Обновление задачи

- `DELETE /api/tasks/{id}/` - Удаление задачи

- `GET /api/tasks/busy-employees/` - Занятые сотрудники

- `GET /api/tasks/important/` - Важные задачи с рекомендациями

### Документация

- `GET /api/schema/` - Схема API

- `GET /api/docs/` - Интерактивная документация (Swagger)

## 🎯 Специальные возможности

### Иерархия задач

Задачи могут иметь родительские и дочерние элементы, что позволяет создавать сложные workflows.

### Важные задачи

Система автоматически определяет задачи, которые:

- Не взяты в работу

- Имеют дочерние задачи, которые уже в работе

- Предоставляет рекомендации по назначению на наименее загруженных сотрудников

### Анализ загруженности
 
Отслеживание количества активных задач на каждого сотрудника для равномерного распределения нагрузки.
